# 提单系统需求规格说明书

## 1. 文档信息
- **文档名称**：提单系统需求规格说明书
- **版本**：V1.0
- **编写日期**：2026-04-17
- **适用平台**：macOS（苹果笔记本电脑）单机版应用
- **数据来源**：`需求表.xlsx` + 补充需求清单

## 2. 项目目标
构建一个在 macOS 本地运行的提单管理工具，支持列表内直接编辑、字段联动校验、任务名规则化生成、筛选查询以及按模板导出 Excel。

## 3. 范围说明
### 3.1 In Scope（本期范围）
- 提单列表展示与行内编辑
- 主题切换（深色/浅色）
- 字段联动与互斥校验
- 任务名正反向联动解析
- 模糊查询
- 按 Excel 模板导出勾选数据

### 3.2 Out of Scope（本期不做）
- 多人协同与在线部署
- 云端账号体系
- 移动端适配

## 4. 使用场景
- **使用者**：个人用户（单用户）。
- **场景**：个人在本机完成提单维护、查询筛选与 Excel 导出，不涉及多角色分工与审批流程。

## 5. 功能需求

### FR-01 主题切换
- 页面整体支持**深色/浅色**两套主题。
- 默认主题可配置（建议默认浅色）。
- 主题切换后即时生效，且记住用户上次选择。

### FR-02 列表直接编辑
- 取消“新建提单”按钮。
- 在列表中直接新增/编辑行数据（支持行内编辑模式）。
- 编辑后可保存，保存前执行字段校验。

### FR-03 省市与排除省市互斥
- `省份/地市` 与 `排除省份/排除地市` 为互斥关系。
- 同一条记录中：
  - 当 `省份` 或 `地市` 有值时，`排除省份`、`排除地市` 进入不可编辑状态。
  - 当 `排除省份` 或 `排除地市` 有值时，`省份`、`地市` 进入不可编辑状态。
- 互斥通过编辑态实时控制，不依赖保存时校验。

### FR-04 码值与名称映射
- `运营商`、`类型` 等字段采用**码值-名称映射**维护。
- 数据保存时统一存储码值。
- 列表显示时展示码值对应名称。
- 任务名中展示码值（不展示名称）。
- 支持名称到码值、码值到名称双向转换，用于任务名正反向解析。

### FR-05 隐藏日期与提示列
- 列表中隐藏以下列：
  - `报单日`
  - `停单日`
  - `应出提示`
- 隐藏列不参与用户日常编辑和显示。

### FR-06 省市基础数据
- 提供中国境内行政区基础数据：
  - 省级行政区（省/自治区/直辖市/特别行政区）
  - 地级市数据
- 用于下拉选择、联动识别与任务名解析。

### FR-07 URL 多值输入与显示
- `URL` 支持输入多个地址，按换行分隔。
- 列表中默认仅展示第一个 URL（其余可在编辑态查看完整内容）。
- 导出时是否包含 URL 由“是否打印URL”控制。

### FR-08 Excel 对齐规则
- 导出 Excel 中：
  - `任务名` 列按原样（不强制居中）。
  - 其他所有列统一水平、垂直居中。

### FR-09 列表固定宽度与横向滚动
- 列表采用固定总宽度（按字段设计宽度展示，不随窗口自动压缩）。
- 当屏幕或窗口宽度不足以完整展示全部列时，显示类似 Excel 的横向滚动条。
- 用户可通过横向滚动查看被遮挡列，列宽与表头对齐保持不变。

### FR-10 打印导出（模板）
- “打印”功能定义为：基于 Excel 模板导出**勾选行数据**。
- 导出文件名规则：`客户名_YYYYMMDD.xlsx`。
- 支持一次勾选多条记录导出。

### FR-11 任务名模糊查询
- 提供任务名关键字模糊搜索。

### FR-12 打印URL开关改造
- 将“打印勾选”与“带URL打印勾选”两个选项合并为单一控件：`是否打印URL`（checkbox）。
- 逻辑：
  - 未勾选：导出内容不包含 URL 信息。
  - 勾选：导出内容包含 URL 信息。
  - 在checkbox下面用文案描述逻辑，防止用户勾选错误

### FR-13 隐藏操作列
- 列表隐藏“操作列”（如编辑、删除按钮列）。
- 对应行为通过行内编辑和外部统一操作实现。

### FR-14 列表字段显示范围
- 列表仅显示以下字段（按顺序）：
  1. 任务名
  2. 类型
  3. 运营商
  4. 行业编码
  5. URL
  6. 数量
  7. 时长
  8. 年龄上限
  9. 年龄下限
  10. pv
  11. 省份
  12. 排除省份
  13. 地市
  14. 排除地市

### FR-15 任务名组成与双向联动
- 任务名由 7 部分组成：
  - `客户名 + 省份名/地市名/全国/三省/四市 + 运营商编码 + 类型编码 + 行业编码 + "-" + 产品名`
- 规则说明：
  1. 省份名/地市名片段与 `省份`、`地市` 字段双向联动。
  2. `运营商编码` 与 `运营商` 字段双向联动（列表显示名称，任务名显示码值）。
  3. `类型编码` 与 `类型` 字段双向联动（列表显示名称，任务名显示码值）。
  4. `行业编码` 与 `行业编码` 字段双向联动。
  5. `省份` 或者 `地市` 与 `省份` 或者 `地市`字段双向联动。（全国或者两省或者三市时要清空`省份` 或者 `地市`字段）
  6. 修改任务名时需可反向解析并回填对应字段；修改字段时任务名同步更新。
  7.  `省份`选择两个以上省份时，显示几个省，比如，选择两个省份时，任务名中显示两省
  8.  `地市`选择两个以上地市时，显示几个市，比如，选择三个地市时，任务名中显示三市
### FR-16 单机版部署要求
- 系统以 macOS 单机版形态交付并运行。
- 无网络时要保证全部功能正常使用。

## 6. 字段规格
| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 任务名 | string | 是 | 按 FR-15 规则生成并支持反向解析 |
| 类型 | enum(code+name) | 是 | 保存码值、显示名称，任务名使用码值 |
| 运营商 | enum(code+name) | 是 | 保存码值、显示名称，任务名使用码值 |
| 行业编码 | string | 是 | 与任务名行业编码段联动 |
| URL | text(multiline) | 否 | 支持多行，列表仅显示首行 |
| 数量 | number/string | 否 | 业务数量 |
| 时长 | enum | 否 | 一天/本周/长期 |
| 年龄上限 | number | 否 | 年龄上限 |
| 年龄下限 | number | 否 | 年龄下限 |
| pv | number/string | 否 | pv 指标 |
| 省份 | multi-select | 否 | 与地市互斥策略按规则执行 |
| 排除省份 | multi-select | 否 | 与省份/地市互斥校验 |
| 地市 | multi-select | 否 | 与省份/排除字段互斥校验 |
| 排除地市 | multi-select | 否 | 与省份/地市互斥校验 |

## 7. 业务规则
- `省份` 与 `地市` 不并存，二者二选一；为空表示全国。
- `省份/地市` 与 `排除省份/排除地市` 通过编辑/不可编辑模式实现互斥，不允许同时编辑填写。
- 多省/多市在任务名中使用数量表达（如“三省”“四市”）。
- 若任务名地域片段为“全国”，则省份与地市字段应为空。

## 8. 查询与筛选
- 支持任务名模糊查询。
- 支持勾选行批量导出。
- 保留基础筛选能力（按客户/类型/运营商等可扩展）。

## 9. 导出规格（Excel）
- 导出来源：当前勾选记录。
- 导出模板：固定 Excel 模板。
- URL 输出策略：由 `是否打印URL` 控制。
- 文件名：`客户名_YYYYMMDD.xlsx`。
- 样式规则：
  - 除 `任务名` 外其余列全部居中。
  - 支持多 URL 换行存储；列表显示首项，不影响导出完整性（当允许导出 URL 时）。

## 10. UI/交互要求
- 支持浅色/深色主题切换。
- 列表固定宽度展示；窗口不足时显示横向滚动条（Excel 风格）。
- 隐藏列：`报单日`、`停单日`、`应出提示`、`操作列`。
- 列表展示字段严格按 FR-14 顺序。

## 11. 非功能需求
- **平台**：macOS 单机运行。
- **可用性**：主流程（编辑-查询-导出）无需联网。
- **性能**：常规数据量下列表编辑与筛选响应流畅。
- **可维护性**：码值字典、省市库、Excel 模板可配置替换。

## 12. 验收标准（UAT）
- 可在同一页面完成行内新增/编辑，无“新建提单”按钮。
- 深浅色主题可切换并记忆用户选择。
- 省市与排除省市互斥通过编辑态实时生效：一侧有值时另一侧自动不可编辑。
- 任务名与类型/运营商/行业编码/地域字段可双向联动。
- 列表字段与顺序与 FR-14 完全一致，且指定列隐藏。
- URL 支持多行输入，列表仅显示第一条。
- 导出文件名符合 `客户名_YYYYMMDD.xlsx`，并可按“是否打印URL”控制 URL 列输出。
- 导出 Excel 对齐规则符合“任务名列除外，其余居中”。
- 列表在小窗口下出现横向滚动条，且滚动后表头与数据列对齐正常。

## 13. 确认项
- 客户名字段是否独立存储，或仅从任务名中解析。
  回答：从任务名中解析
- 任务名地域片段与“多省/多市”数量阈值规则（如两省是否显示“两省”）。
  回答：是的
- 运营商、类型、行业编码的完整码表来源与维护方式。
  回答：内置到系统中
- Excel 模板文件路径、模板版本控制方式。
  回答：template.xlsx
- 开发框架用 Python + PySide6
